# Architecture

High-level pipeline and HSL lab topology are in [Architecture diagrams](diagrams.md) (Mermaid flowcharts). For a single narrative of how the pieces fit together (layering, env construction, step flow, golden vs benchmark), see [System overview](system_overview.md).

LabTrust-Gym is structured as:

- **Core simulator (`engine/`)** — `core_env` (reset, step, query), audit log (hash chain, forensic freeze), zones (graph, doors, device placement), specimens (acceptance, hold/reject), QC (result gating), critical (notify/ack), queueing (per-device queues), devices (state machine, service times when `timing_mode: simulated`), clock, RNG (deterministic), **signatures** (Ed25519 verification; key registry with lifecycle: status ACTIVE/REVOKED/EXPIRED, not_before_ts_s/not_after_ts_s; SIG_KEY_REVOKED, SIG_KEY_EXPIRED, SIG_KEY_NOT_YET_VALID; agent_id binding), **runtime control** (UPDATE_ROSTER, INJECT_SPECIMEN: always require SYSTEM + R_SYSTEM_CONTROL key and signature; control_decision in step output and episode log), **transport** (multi-site consignments; sites_policy routes and temp drift), **invariants_runtime** (registry-driven checks post-step), **enforcement** (policy-driven throttle/kill_switch/freeze_zone/forensic_freeze). BLOCKED steps do not mutate world state except audit logging; hash chain break triggers forensic freeze. At reset, policy values are resolved via `engine/policy_resolution.py`: effective_policy over file (policy_root) over default.
- **Trust skeleton (`policy/`)** — Versioned YAML/JSON under `policy/` (emits vocab, zones, **sites** (sites_policy.v0.1: sites, site graph, routes, transport_time, temp_drift), reason codes, tokens, invariants registry v1.0, catalogue, stability, equipment, critical, enforcement map, studies, llm action schema, golden scenarios). All policy files are validated against JSON schemas in `policy/schemas/` via `labtrust validate-policy`.
- **Runner** — **Golden runner** (`runner/golden_runner.py`): runs scenarios from `policy/golden` against a `LabTrustEnvAdapter` (no PettingZoo); asserts step contract and emit vocabulary (correctness backbone). **Benchmark runner** (`benchmarks/runner.py`): builds env via domain adapter → CoreEnv → `LabTrustParallelEnv`, drives the step loop with baselines/coordination/risk injectors, writes results and logs. See [System overview](system_overview.md#golden-runner-vs-benchmark-runner).
- **Envs (`envs/`)** — PettingZoo Parallel and AEC wrappers over the core engine; the benchmark runner constructs the env by wrapping the domain adapter's `LabTrustEnvAdapter` (e.g. CoreEnv) in `LabTrustParallelEnv`. Observations, actions, rewards, infos (require `.[env]`).
- **Baselines (`baselines/`)** — Scripted ops, scripted runner, adversary, **LLM agent** (constrained decoder, rationale required, **DeterministicConstrainedBackend** as official baseline; **canonical allowed-actions payload** in `baselines/llm/allowed_actions_payload.py`; **ProviderBackend** protocol and **OpenAILiveBackend** for live runs; **LLM_DECISION** audit emit and `_llm_decision` payload in episode log/evidence bundle; mock/deterministic and live backends), PPO/MARL (optional `.[marl]`).
- **Benchmarks (`benchmarks/`)** — Task definitions (throughput_sla, stat_insertion, qc_cascade, adversarial_disruption, **multi_site_stat**, **insider_key_misuse** (5 attack phases: forbidden action, forged sig, replay, revoked key → SIG_KEY_REVOKED, token misuse)), metrics, runner; CLI `labtrust quick-eval`, `labtrust run-benchmark` (optional `--llm-backend deterministic|openai_live`; see [Live LLM](../agents/llm_live.md)), `labtrust bench-smoke`.
- **Studies (`studies/`)** — Study runner (ablations → conditions), plots (figures + data tables), reproduce (minimal sweep + plots); CLI `labtrust run-study`, `labtrust make-plots`, `labtrust reproduce --profile minimal | full`, `labtrust package-release`.

**CLI:** Representative commands: `--version`, `validate-policy`, `validate-fhir` (bundle + optional terminology value-set validation), `quick-eval`, `run-benchmark`, `eval-agent`, `bench-smoke`, `run-study`, `make-plots`, `reproduce`, `export-receipts`, `export-fhir`, `verify-bundle`, **`ui-export`** (UI-ready zip from run dir), `package-release`, `summarize-results`, `generate-official-baselines`, `run-official-pack`, `run-security-suite`, `export-risk-register`, `validate-coverage`, `build-release-manifest`, `verify-release`, `run-coordination-security-pack`, `build-lab-coordination-report`, `determinism-report`, `train-ppo`, `eval-ppo`. For the full list run `labtrust --help` or see [CLI contract](../contracts/cli_contract.md). Policy path: resolved via `config.get_repo_root()` (LABTRUST_POLICY_DIR, package data, or repo). See [UI data contract](../contracts/ui_data_contract.md) for ui-export bundle format.

Determinism: single RNG wrapper (`rng.py`) seeded from scenario; no ambient randomness. Timing: `timing_mode: explicit` (default, event `t_s` only) or `simulated` (device service times from equipment registry).
